## Overview

First week of OJT at Makerspace Innovhub. Got oriented with the company culture and technical workflows, then immediately dove into real project work on PandanChain.

## Day-by-Day

**Feb 9** — Attended the company orientation and rules briefing. Went through the interview process for Front-End and Back-End Web Development positions. Was assigned as **Front-End Developer** for the Document Management System project, internally branded as **PandanChain**.

**Feb 10** — Studied GitHub workflows and version control best practices. Reviewed the PRD (Product Requirements Document) for the DMS project to understand scope and technical expectations.

**Feb 11** — Designed and built the first draft of the **marketing landing page** for PandanChain. Submitted an unfinished draft for review to get early feedback.

**Feb 12** — Continued landing page development based on review feedback. Applied major revisions and pushed the **final output** to the team repository.

**Feb 13** — Explored **Node.js** fundamentals through curated video tutorials shared on the team Discord channel.

## Key Learnings

- GitHub branching and pull request workflows in a team environment
- Reading and interpreting Product Requirements Documents (PRD)
- Iterative design: shipping a draft early, receiving feedback, and refining

## Challenges

Getting used to the team's Git workflow took some adjustment, especially structuring commits clearly and working from a shared repository without conflicts.
